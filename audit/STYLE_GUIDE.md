# CLB-4 BFR Cosmetic Style Guide

This is the binding visual specification for any BFR workbook generated by
the pipeline. It is extracted from the four rebuilt clean CCN sheets in
`SW_M29030_CLB4_BFR_2026-NWPCW167400L021.xlsx` (`14345`, `21451`, `21455`,
`61072`) — full per-sheet fingerprints in
`audit/reports/12_style_clb4_clean_ccns.txt`.

The output must reproduce this look. No theme drift, no "openpyxl default"
artifacts, no arbitrary palette choices.

## Colors

The workbook uses theme-derived colors, not raw RGB. Targeting the
default Office theme (Calibri / "minor" scheme). Recurring fills:

| Use | Fill spec |
|---|---|
| Title banner (A1:AF2) | no fill (white) |
| Section header banner (A6:AF6) | `theme=1, tint=-0.1499984740745262` (≈ light grey, 15 % darker than white) |
| Sub-header / label cells (A7, A15, AB15, AB16) | `theme=1, tint=-0.1499984740745262` solid |
| Column-header cells (V7, Y7, AB7) | `theme=1, tint=-0.0499893185216834` solid (≈ very light grey, 5 %) |
| Body cells | no fill |

All borders are **thin, color indexed=64** (auto/black). The body data
table uses thin borders on all 4 sides; the header / label cells inherit
the same.

## Merged cell pattern (per-sheet skeleton)

Every CCN sheet shares this skeleton:

```
A1:AF2     — title banner ("BASIC FACILITY REQUIREMENTS WORKSHEET")
A3:E3      — "Installation:" label
F3:L3      — installation name
M3:Q3      — "Tenant:" label
R3:X3      — tenant unit name
A4:E4      — "Installation UIC:" label
F4:L4      — UIC value
M4:Q4      — "Tenant UIC:" label
R4:X4      — UIC value
Y4:AD4     — "Planning Area: <area>"
A5:B5      — "CCN:" label
C5:L5      — CCN value
M5:Q5      — "Nomenclature:" label
R5:AF5     — CCN nomenclature
A6:AF6     — section banner ("SPACE REQUIREMENT ANALYSIS · CCN xxxxx · UFC 2-000-05N …")
A7:U7      — sub-banner / column-block header
V7:X7, Y7:AA7, AB7:AF7   — three numeric column headers (e.g. "SF/Person", "Count", "NSF")
A8:U8, A9:U9, …          — per-row label spans (each row uses one merged label cell A:U)
A15:AA15   — subtotal row
A16:U16    — total NSF label
AB15:AF15, AB16:AF16     — value cells
A37:G37    — "TOTAL REQUIREMENT:" merged
H37:K37    — total value cell
L37:N37    — unit label cell (GSF / GSY)
```

The exact set of merged ranges differs per sheet by ±10 ranges depending
on how many data rows the CCN has, but the skeleton above is invariant.

## Page-end "TOTAL REQUIREMENT" footer

Every CCN sheet ends with the same three-cell pattern at the print-area
bottom (typically row 37, sometimes row 34 or 40 depending on layout):

```
A:G       "TOTAL REQUIREMENT:"        — Calibri 11 bold, top border thin
H:K       =<final result cell>        — Calibri 10, format `#,##0\ "GSF"`
L:N       GSF (or GSY)                — Calibri 11 bold, top border thin
AB        Page X of Y                 — Calibri 11 right-aligned (only on multi-page sheets)
```

This footer is the contract between the CCN sheet and the `UNIT_ROLLUP`
— the rollup pulls H37 (or its equivalent) for that CCN's grand total.

## What NOT to do

- **No bright tab colors** — these sheets all have `tab_color=None`.
  The hidden broken sheets use `FFFF0000` (bright red); never adopt that
  pattern in output.
- **No `IFERROR(…,"")` masking** of formulas. Errors must surface, not
  hide. Round-1 audit confirmed this is how the broken hidden sheets
  silently failed for years.
- **No restricted-range lookups** (e.g. `'TO'!$B$5:$B$309`). Use full
  columns (`'TO'!$B:$B`) so adding rows doesn't break the contract.
- **No external links** — every formula must resolve inside the workbook.
- **No defined names with `#REF!` or `#N/A` values** — pollution.
- **No 2017-era template residue** — the source data is the unit being
  generated for, period.
