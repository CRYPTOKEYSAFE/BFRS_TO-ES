================================================================
Itemized defects, in priority order
================================================================

DEFECT 1 (CRITICAL): CCN 61073 = 0 GSF, 574 billets routed there.
  Cell H40 holds the literal value 0, not a formula.
  574 clinical platoon and Co HQ admin Marines have no facility
  space in the BFR.
  This is the same issue as Apex Omega Violation 2 from commit
  9a01d01. The doctrinally clean fix is adding an FC Series 500
  medical CCN (Dispensary / Outpatient Clinic 53010 plus possibly
  Hospital components) and re-routing the clinical billets out of
  61073 into the Series 500 sheet.
  The lighter fix is to make 61073 H40 a real COUNTIFS x SF formula
  matching the 61072 pattern, accepting that admin space rates are
  not the right doctrinal rate for clinical platoons.

DEFECT 2: CCN 14312 (Operational Vehicle Laydown Area) = 0 GSF.
  The chain reads from TE rows tagged 14312 (8 rows per validator)
  via VLOOKUP for length/width/height columns. Every VLOOKUP in
  the chain returns the string "0" via the IFERROR fallback. The
  IFERROR fallback is wrapped in quotes, so the formula always
  returns "0" rather than 0; mixed string/number multiplication
  collapses to 0 downstream.
  Two issues to verify:
    (a) Are the TE rows actually populated in cols T/U/V/W (the
        length/width/height/volume cols the VLOOKUPs target)?
    (b) Should IFERROR fallback be 0 (number) rather than "0"
        (string)?

DEFECT 3: CCN 45110 (Open Storage Area) = 8.89 GSF.
  Eight square feet of open storage is not credible. The chain:
    M25 = SUM(B25:K25) = 8.8888
    B25 = R32 * 0.11111 = 80 * 0.11111 = 8.8888
    R32 = SUM(R29:T31) = 80 (R29 = 0, so the 80 is in R30 or R31)
  The 0.11111 factor is 1/9 (SF to SY conversion). The output
  unit may be wrong (expressed as SY but labeled GSF), or the
  inputs are wrong, or both.

DEFECT 4: CCN 21451 (Auto Org Shop) = 1,129 GSF.
  N24 = SUM(B24:M24) = 1,129. B24 (Q32 chain) returns 0 because
  the equipment-side calc is empty: M28 (=AB173) = 0. The 1,129
  comes from columns C-M of row 24, not from the documented
  equipment-driven Q32 chain. For 70 motor T billets and an
  organizational shop, 1,129 SF is too low (per-Marine rate would
  be 16 SF, well below typical USMC motor T shop rates).

DEFECT 5: CCN 21710 (Comm Maint Shop) = 583 GSF.
  M37 (officers) = 0 across all UICs. P37 (junior enlisted) = 4.
  Total = 360 SF + ~223 SF from rows 28-30. Likely undercounts
  comm space because H&S has multiple radio/electronics TAMCNs
  on the TE side that are not feeding the equipment-driven chain.

DEFECT 6: CCN 61072 (BN HQ Admin) = 69,875 GSF.
  P30 sum = 69,875. P26 = 39 (BN admin Marines) x 162.5 = 6,337.5.
  Remaining 63,537.5 GSF comes from rows 27/28/29 picking up
  COUNTIFS on the 61073 NOTE for M28262 / M28263. That captures
  clinical platoon billets at the BN HQ admin rate (162.5 SF/Marine),
  which is doctrinally wrong: clinical platoons need clinical
  facility CCNs (Series 500), not admin space.
  This defect is the mirror image of DEFECT 1; both arise from
  the absence of a Series 500 CCN in the workbook.
